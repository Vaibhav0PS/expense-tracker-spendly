     Implementation Plan: Edit Expense Feature

     Context

     This plan implements the "Edit Expense" feature (Step 8) for the Spendly
     expense tracker. Users will be able to edit their existing expenses through
      a form pre-filled with current values, with proper ownership validation to
      ensure users can only edit their own expenses.

     Changes Overview

     1. Database Queries (database/queries.py)

     - Add get_expense_by_id(expense_id, user_id) function to fetch an expense
     only if it belongs to the specified user
     - Add update_expense(expense_id, user_id, amount, category, date, 
     description) function to update an expense with ownership verification
     - Modify get_recent_transactions() to include the id column in the SELECT
     clause so templates can build edit links

     2. Application Logic (app.py)

     - Import the new query functions from database.queries
     - Replace the placeholder /expenses/<int:id>/edit route with a proper
     GET+POST handler
     - GET handler: Fetch expense using get_expense_by_id, return 404 if not
     found/not owned, otherwise render edit form
     - POST handler: Validate form data (same rules as add expense), call
     update_expense, redirect to profile on success
     - Update route decorator to accept both GET and POST methods

     3. Templates

     - Create templates/edit_expense.html: Form template extending base.html
     with fields pre-populated from expense data
     - Modify templates/profile.html: Add "Actions" column header and Edit link
     (<a href="/expenses/{{ tx.id }}/edit">Edit</a>) to each transaction row

     Implementation Steps

     Phase 1: Database Layer

     1. Add get_expense_by_id(expense_id, user_id) to database/queries.py
     2. Add update_expense(expense_id, user_id, amount, category, date, 
     description) to database/queries.py
     3. Modify get_recent_transactions() to include id in SELECT clause

     Phase 2: Application Layer

     1. Import new functions in app.py
     2. Implement GET handler for /expenses/<int:id>/edit:
       - Check authentication
       - Call get_expense_by_id(id, session['user_id'])
       - If None, return 404
       - Otherwise render edit_expense.html with expense data and categories
     list
     3. Implement POST handler for /expenses/<int:id>/edit:
       - Check authentication
       - Verify ownership via get_expense_by_id (or direct check)
       - Validate form data (amount, category, date)
       - Call update_expense with validated data
       - On success: redirect to profile
       - On failure: re-render form with errors and preserved values
     4. Update route decorator to @app.route("/expenses/<int:id>/edit", 
     methods=["GET", "POST"])

     Phase 3: Frontend

     1. Create templates/edit_expense.html based on add_expense.html but with
     pre-filled values
     2. Modify templates/profile.html to add Actions column and Edit links

     Key Functions to Reuse/Reference

     - Existing validation logic from add_expense route in app.py
     - Existing insert_expense function in database/queries.py (as reference for
      update_expense)
     - Existing get_user_by_id function in database/db.py (for ownership
     patterns)
     - Existing form structure in add_expense.html

     Testing Approach

     1. Manual testing:
       - Login as a user
       - Navigate to profile
       - Click Edit on an existing expense
       - Verify form is pre-filled with correct data
       - Submit valid changes and verify they appear in transaction list
       - Test validation errors (invalid amount, category, date)
       - Verify unauthorized access returns appropriate responses
     2. Automated tests (to be written later):
       - Unit tests for new query functions
       - Route tests for GET/POST endpoints
       - Security tests for ownership verification

     File References

     - database/queries.py - Add query functions
     - app.py - Implement route handlers
     - templates/edit_expense.html - Create new template
     - templates/profile.html - Modify transaction table
